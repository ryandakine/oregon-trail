# Mobile App Plan: Oregon Trail AI

## Recommended Approach: PWA + TWA (Android First)

**Why:** Android has ZERO competition. TWA gets us into Google Play fastest with minimal code changes. The game is already responsive (45ch mobile). PWA gives installability on both platforms. TWA wraps the PWA for a Play Store listing.

**Timeline:** 4-6 hours to PWA + TWA. Ship to Play Store within days.

## Phase 1: PWA (2-3 hours)

### 1.1 Service Worker (`public/sw.js`)

Cache-First for static assets, Network-Only for API:

```javascript
const CACHE_NAME = 'ot-v1';
const STATIC_ASSETS = [
  '/', '/index.html', '/game.js', '/ui.js', '/style.css',
  '/newspaper.js', '/tombstone.js', '/manifest.json',
  '/icons/icon-192.png', '/icons/icon-512.png'
];

self.addEventListener('install', e => {
  e.waitUntil(caches.open(CACHE_NAME).then(c => c.addAll(STATIC_ASSETS)));
});

self.addEventListener('fetch', e => {
  if (e.request.url.includes('/api/')) return; // network-only for API
  e.respondWith(
    caches.match(e.request).then(r => r || fetch(e.request))
  );
});
```

## Phase 2: Mobile Enhancements (1-2 hours)

### 2.1 Web Share API (native share sheet)

Replace download-only sharing with native share when available:

```javascript
// In tombstone.js and newspaper.js, after html2canvas:
canvas.toBlob(async (blob) => {
  const file = new File([blob], 'oregon-trail.png', { type: 'image/png' });
  if (navigator.canShare?.({ files: [file] })) {
    await navigator.share({
      title: 'Oregon Trail — AI Edition',
      text: 'My journey on the Oregon Trail',
      files: [file],
      url: 'https://trail.osi-cyber.com'
    });
  } else {
    // Fallback to download link (current behavior)
  }
});
```

### 2.2 Haptic Feedback (Android)

Add to ui.js at key moments:

```javascript
// Death
navigator.vibrate?.(200);

// Disease contracted
navigator.vibrate?.([100, 50, 100]);

// Critical supplies (food = 0)
navigator.vibrate?.([50, 30, 50, 30, 50]);

// Party wipe
navigator.vibrate?.([300, 100, 300]);

// Agency steal (horror tier)
navigator.vibrate?.([50, 50, 50, 50, 200]);
```

Zero-cost (one line each, no-op on iOS via optional chaining).

## Phase 3: TWA for Google Play (1-2 hours)

### 3.1 Prerequisites
- Google Play Console developer account ($25 one-time fee)
- PWA passing Lighthouse audit (service worker + manifest + HTTPS)
- Digital Asset Links file (`.well-known/assetlinks.json`) on the domain

### 3.2 Build with PWABuilder or Bubblewrap

**Option A: PWABuilder (easiest)**
1. Go to https://www.pwabuilder.com
2. Enter https://trail.osi-cyber.com
3. Click "Package for stores" → Android
4. Download the generated APK/AAB
5. Upload to Google Play Console

**Option B: Bubblewrap CLI**
```bash
npm i -g @nicepkg/nicepkg-core @nicepkg/nicepkg-cli
npx @nicepkg/nicepkg-cli init --manifest https://trail.osi-cyber.com/manifest.json
npx @nicepkg/nicepkg-cli build
```

### 3.3 Digital Asset Links

Create `public/.well-known/assetlinks.json`:
```json
[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "com.osicyber.oregontrail",
    "sha256_cert_fingerprints": ["YOUR_SIGNING_KEY_FINGERPRINT"]
  }
}]
```

The fingerprint comes from the signing key generated during the build process.

## Phase 4: iOS (deferred)

iOS options after Android validates:

1. **PWA on iOS:** Already works (Add to Home Screen). No App Store listing but functional.
2. **App Store via Capacitor:** Wrap in Capacitor for native iOS shell. Apple requires additional review for web-view apps but games are generally accepted.
3. **App Store via PWABuilder:** Generate an iOS package (uses WKWebView).

**iOS-specific considerations:**
- Apple charges $99/year for developer account (vs $25 one-time for Google)
- Apple App Store review is stricter (especially for web-view wrappers)
- No `navigator.vibrate()` on iOS
- Web Share API works on iOS 15+ with files
- Push notifications require Apple Push Notification service (APN)

**Recommendation:** Ship Android first, validate traction, then iOS if it grows.

## Cost Considerations at Scale

Current: ~$0.07-0.08 per run (Haiku 4.5).

| Scale | Monthly Cost | Mitigation |
|-------|-------------|------------|
| 1K plays/mo | $70-80 | None needed |
| 10K plays/mo | $700-800 | Tiered model routing (Workers AI Llama for routine) |
| 100K plays/mo | $7-8K | Kill switch + cache-only mode after daily budget |
| 1M plays/mo | $70-80K | PROBLEM. Need sponsorship, donations, or premium tier. |

**At app store scale:** If the app gets 10K+ downloads, costs become real. Plan:
1. Workers AI Llama 3.1 8B for routine events (free tier)
2. Haiku only for landmarks, deaths, newspapers
3. Daily budget cap with graceful degradation to cached events
4. Consider a "Premium" tier ($2.99 one-time) for unlimited Haiku events
